Enumerable

Using partition to create an array of the items you want and a separate array for what's leftover

1/31/2016

The Enumerable class is an extremely powerful one in Ruby. This class gives you several ways to iterate through a collection of items and temporarily hold them until you are ready to change the orginal collection or create a brand new collection.

The partition method will return an array containing items meeting the criteria you defined and a second array containing the items that did not meet your criteria. This can greatly simplify solutions that require you to divide one collection into many. Here's an example of how you might be doing this today without the partition method. In the example below we want to take a hash of names and ages and break them into two collections. One collection should contain the people with ages less than the one we define. The rest should go into a separate collection:

def group_method(name_age_hash, age)
below_age = name_age_hash.select { |k,v| v <= age}.to_a.compact
above_age = name_age_hash.select { |k,v| v > age}.to_a.compact
output = below_age, above_age
end

Note how we're repeating ourselves in the above example when trying to divide the array into two collections. We're also using a step to combine the two arrays at the end. Here's how you could do the same thing using the partition method:

def my_hash_splitting_method(source, age)
source.partition { |k,v| v <= age}.to_a
end

This will accomplish the same objective with much simpler and DRY code.
